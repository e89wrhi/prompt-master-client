# Template: Professional Product Description Writing

Use this template to transform dry product specs into compelling, benefit-led stories that drive sales. It is designed to move beyond "What the product is" and into "What the product does for the user," utilizing sensory language, emotional triggers, and optimized formatting.

## 📋 Prompt Template

```text
Act as a Senior eCommerce Copywriter and Product Storyteller. Write a [LENGTH] product description for [PRODUCT_NAME].

### Product Specs:
[INSERT_TECHNICAL_SPECS_AND_MATERIALS]

### Writing Requirements:
1. **The "Benefit-Head":** Open with 1-2 punchy sentences that state the [PRIMARY_TRANSFORMATION] (e.g., "Feel lighter with every step").
2. **Sensory Storytelling**: Describe the [TEXTURE/SCENT/WEIGHT/LOOK] in a way that allows the user to imagine owning the product.
3. **The "Feature-to-Benefit" Table**: Translate 3 technical specs into human benefits (e.g., "Gore-Tex Fabric" -> "Stays dry in any storm").
4. **Ideal Use Case**: Describe a specific [MOMENT_IN_TIME] where this product is the hero (e.g., "The perfect morning ritual...").
5. **Technical "Bullet Points"**: Provide a clean, organized list of secondary specs for the [LOGICAL_BUYER].

### Output Constraints:
- Use a [TONE] tone.
- Ensure the description is optimized for [PLATFORM] (e.g., Amazon, Shopify, Etsy).
- Include 3 SEO-friendly keywords organically within the text.
```

## 🧩 Variables to Fill Out

- `[PRODUCT_NAME]` - The item.
- `[INSERT_TECHNICAL_SPECS]` - Raw data (e.g., 100% cotton, 4000mAh battery).
- `[PRIMARY_TRANSFORMATION]` - How the user moves from "Before" to "After."
- `[MOMENT_IN_TIME]` - A specific scene where the product is used.
- `[TONE]` - e.g., Luxurious/Elegant, Rugged/Durable, Fun/Playful.
- `[PLATFORM]` - Defines the formatting (Amazon needs long bullets, Shopify needs a nice flow).

## 💡 Pro-Tips

- **The "Who Cares?" Rule:** Every feature must be followed by "Which means..." (e.g., "It has a 10-hour battery, *which means* you can work all flight without hunting for an outlet").
- **Scannability is Speed:** People don't read product descriptions; they scan them. Keep paragraphs under 3 lines and use bold text for key terms.
- **Answer the "Why Not?":** If the product is expensive, justify the price with [CRAFTSMANSHIP_DETAILS]. If it's cheap, justify it with [EFFICIENCY_SCALING].
