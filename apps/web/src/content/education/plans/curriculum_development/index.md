# Template: Strategic Curriculum Development

Use this template to design a high-level educational roadmap for a long-form course or training program. It is designed to bridge the gap between "A list of topics" and a "Structured Learning Path" by utilizing Bloom's Taxonomy, clearly defined learning outcomes, and a logical progression of difficulty.

## 📋 Prompt Template

```text
Act as a Dean and Senior Curriculum Architect. Develop a [DURATION] Curriculum for [COURSE_TOPIC].

### Course Context:
- **Target Learner**: [e.g., Beginners with no code experience, Senior Managers, Undergraduate Students].
- **Primary Goal**: [e.g., Mastery of React, Mastering Leadership, Understanding Microeconomics].
- **Final Outcome**: [WHAT_THE_LEARNER_WILL_BE_ABLE_TO_DO_AT_THE_END].

### Curriculum Requirements:
1. **The Learning Objectives**: Provide 3-5 objectives using Bloom's Taxonomy verbs (e.g., "Analyze...", "Synthesize...", "Evaluate...").
2. **Modular Architecture**: Break the curriculum into [NUMBER_OF_MODULES] distinct modules. For each module, include:
   - **Theme**: The core focus (e.g., "The Foundations," "Advanced Integration").
   - **Key Lessons**: A list of 3-5 sub-topics covered.
   - **The Hands-on Project**: A practical application task for that specific module.
3. **Assessment Strategy**: Describe how the learner's progress will be measured (e.g., Quizzes, Peer-reviews, Final Capstone Project).
4. **Prerequisite Knowledge**: Define exactly what the learner *must* know before starting Module 1.
5. **Recommended Resources**: Suggest 3 types of [SUPPLEMENTARY_MATERIALS] (e.g., case studies, specific software, reading list).

### Output Constraints:
- Use a Professional, Academic, and Structured tone.
- Present the curriculum in a detailed Table or Outline format.
- Ensure the progression is "Scaffolded"—meaning each module builds on the previous one.
```

## 🧩 Variables to Fill Out

- `[COURSE_TOPIC]` - The subject.
- `[DURATION]` - e.g., 12-week Intensive, 4-hour Workshop, 6-month Bootcamp.
- `[NUMBER_OF_MODULES]` - Usually 4-10 depending on duration.
- `[SUPPLEMENTARY_MATERIALS]` - Books, videos, coding sandboxes.
- `[FINAL_OUTCOME]` - The "Hero Product" the student builds.

## 💡 Pro-Tips

- **The "Scaffolding" Principle:** Don't give them the hard stuff first. Ask the AI to "Check the cognitive load of each module to ensure it doesn't spike too early."
- **Backward Design:** Start with the end in mind. Ask the AI to "Reverse-engineer the curriculum from the Final Capstone project back to Module 1."
- **Industry Alignment:** If the course is for professionals, ask the AI to "Include a section on how these skills map to current [JOB_MARKET_TRENDS]."
