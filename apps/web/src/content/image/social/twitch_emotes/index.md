# Template: High-Visibility Twitch & Discord Emotes

Use this template to generate small-scale, high-expression emotes for streaming platforms. It is designed to overcome the "Tiny Resolution" challenge by utilizing [EXAGGERATED_MESA_EXPRESSIONS], [HEAVY_BLACK_OUTLINES], and [FLAT_VIBRANT_COLORS] to ensure your emotes are readable even at 28x28 pixels in a fast-moving chat.

## 📋 Prompt Template

```text
Act as a Streamer Brand Designer and Emote Artist. Generate a prompt for a set of [NUMBER_OF_EMOTES] [CHARACTER_OR_MEME] emotes.

### Emote Context:
- **Character Base**: [e.g., A cute cat, a futuristic robot, a stylized version of the founder].
- **Style**: [e.g., Kawaii Chibi, Bold 90s Cartoon, Clean Vector, Pixel Art].

### Visual Requirements:
1. **The Exaggerated Expression**: Define the 3 specific [EMOTIONS] (e.g., HYPE: Arms in the air; GG: Golden crown on head; SHOCK: Jaw dropped to the floor).
2. **Readability Guardrails**: Every emote MUST have a [HEAVY_DARK_OUTLINE] to separate it from the chat background. Use [FLAT_COLORS] with minimal gradients.
3. **The "Face-Focus" Rule**: Describe the character's head as taking up [80_PERCENT_OF_THE_CANVAS]. Features like eyes and mouths should be [DOUBLE_SIZED].
4. **Color Palette**: Use [VIBRANT_PRIMARY_COLORS] that pop against both "Dark Mode" and "Light Mode" UIs.
5. **Technical Specifier**: Conclude with fidelity cues (e.g., "Sharp vector edges, no shadows, transparent background look, high contrast, clean cell-shading").

### Output Constraints:
- Use a Fun, High-Energy, and Icon-Focused tone.
- Avoid intricate background details; focus entirely on the [EXPRESSION].
- Provide the final prompt in a copy-pasteable code block.
```

## 🧩 Variables to Fill Out

- `[CHARACTER_OR_MEME]` - Who is the emote personifying?
- `[NUMBER_OF_EMOTES]` - e.g., 3, 5, or 10.
- `[EMOTIONS]` - e.g., Love (Heart eyes), Anger (Red face), LUL (Laughing).
- `[KAWAII / CHIBI / etc]` - The specific art style.

## 💡 Pro-Tips

- **The "White Background" Test:** Always ask for a [SOLID_WHITE_BACKGROUND]. This makes it easier to remove the background and create a transparent .png for the bot/platform.
- **Eyes and Mouths are the Story:** In a tiny emote, the body doesn't matter. Focus the prompt entirely on the [FACIAL_REACTION].
- **Symbol Clues:** Use universal symbols next to the face to reinforce the feeling (e.g., [Zzz] for sleeping, [$$$] for profit, [!] for surprise).
