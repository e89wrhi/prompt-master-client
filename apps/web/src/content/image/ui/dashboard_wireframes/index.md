# Template: Strategic Dashboard Wireframes

Use this template to generate clean, high-concept wireframes for data-heavy applications. It is designed to visualize [LAYOUT_STRUCTURES]—such as sidebars, top navigation, and complex information grids—utilizing a [LOW-FIDELITY_GRAYSCALE] or [BLUEPRINT_STYLE] to ensure the focus remains on user experience (UX) and data hierarchy rather than aesthetic colors.

## 📋 Prompt Template

```text
Act as a Product Manager and Senior UX Designer. Generate a high-concept wireframe prompt for a [DASHBOARD_TYPE] dashboard.

### UX Objective:
- **Core User Goal**: [e.g., Monitoring sales performance, managing a server fleet, tracking personal health metrics].
- **Data Density**: [e.g., Clean and spacious, high-density/info-heavy, mobile-optimized].

### Layout Requirements:
1. **The Navigation Architecture**: Describe the [SIDE_OR_TOP_NAV] (e.g., "A collapsible left-hand sidebar with 5 clear icons," "A floating minimal top navigation bar").
2. **Main Content Grid**: Define the [BLOCK_STRUCTURE] (e.g., "A 3-column layout where the center column is the largest," "A bento-box grid of variable-sized cards").
3. **Data Visualization Blocks**: Suggest specific [PLACEHOLDER_ELEMENTS] (e.g., "Large bar chart in the top right," "A list of recent transactions at the bottom," "Three high-level KPI cards at the top").
4. **The Visual Style**: Specify a [LOW-FIDELITY_GRAYSCALE] or [TECHNICAL_BLUEPRINT] aesthetic (e.g., "Clean gray and white boxes," "Minimalist thin black lines on an off-white background").
5. **Interactive Markers**: Briefly describe [UI_CONTROLS] (e.g., "Search bar in the header," "Filter dropdowns on the right," "A giant 'New Project' button").

### Output Constraints:
- Use a Precise, Logical, and Structural tone.
- Avoid colors or high-detail imagery; focus on [HIERARCHY_AND_LAYOUT].
- Provide the final prompt in a copy-pasteable code block.
```

## 🧩 Variables to Fill Out

- `[DASHBOARD_TYPE]` - e.g., Fintech, eCommerce Admin, HR Portal, Smart Home Hub.
- `[BLOCK_STRUCTURE]` - How the pieces of the puzzle fit together.
- `[LOW-FIDELITY / BLUEPRINT]` - The visual language of the wireframe.
- `[HIERARCHY_AND_LAYOUT]` - The soul of the design.

## 💡 Pro-Tips

- **Bento Grids are for Metrics:** If your dashboard has many small, unrelated data points, use a [BENTO_BOX_LAYOUT]. It organizes different "Information Tiles" into a cohesive but varied grid.
- **The "F-Pattern" Rule:** Users read from [TOP-LEFT_TO_BOTTOM-RIGHT]. Put your most important KPI (e.g., Total Revenue) in the [TOP_LEFT_SECTION]. Ask the AI to "Optimize for a standard F-pattern scanning journey."
- **Grayscale clarifies Value:** By removing color, you force the AI to distinguish elements through [CONTRAST_AND_WEIGHT] (e.g., "Bold dark gray header against light gray background").
