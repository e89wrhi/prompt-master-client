# Examples: Stakeholder Update Emails

## 1. Project is "Amber" (Minor Delay)

```text
Act as a Senior Project Manager at a global technology firm. Draft a stakeholder update email regarding the "Mobile App Refresh" for the period of Week of May 12th.

### Project Status:
- Wins: Finished the high-fidelity mockups; Backend API for login is 100% stable.
- Losses: The senior UI designer is on sick leave for 3 days; delayed the "Profile Page" design.
- Feedback: Version 0.8 test build was "slightly sluggish" on older iPhones.

### Email Requirements:
1. **Subject Line**: Write 3 options.
2. **Current Status (RAG)**: Classify as Amber due to design delays.
3. **Key Milestones Achieved**: Mockups finished, Login API stable.
4. **Current Blockers**: Design bottleneck due to resource leave.
5. **Upcoming Priorities**: Finish Profile Page, Optimize performance for older devices.
6. **Timeline Health**: 3 days behind "Ideal" schedule, but still on track for June launch.

### Output Constraints:
- Use a Transparent and Proactive tone.
- Include a "Recovery Plan" to make up the 3 lost days.
```

## 2. Project is "Green" (On Track)

```text
Act as a Senior Project Manager. Project: Q3 Compliance Audit.

### Project Status:
- Wins: All 40 interviews finished; initial data collection done.
- Blockers: None.
- Timeline: Ahead of schedule.

### Email Requirements:
1. **Current Status (RAG)**: Green.
2. **Key Milestones Achieved**: Internal data gathering complete.
3. **Upcoming Priorities**: Draft the final report, Schedule the board review.
4. **Timeline Health**: 1 week ahead of schedule.

### Output Constraints:
- Use a Professional and Confident tone.
```
