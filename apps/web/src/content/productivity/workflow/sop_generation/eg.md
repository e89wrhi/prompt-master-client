# Examples: SOP Generation

## 1. Deploying a Website (Netlify/GitHub)

```text
Act as a Senior Systems Manager. Generate a formal SOP for "Deploying the Staging Branch to Production."

### Source Material:
I check the staging URL first. If it looks good, I go to GitHub and create a Pull Request. I wait for the CI to pass. Then I merge it. Then I check the Netlify dashboard to see if the build finished.

### SOP Requirements:
1. **Objective**: Safe deployment of code.
2. **Prerequisites**: Access to GitHub 'Main' branch and Netlify Admin.
3. **The Step-by-Step Procedure**:
   - Phase 1: Validation (Staging check).
   - Phase 2: Deployment (PR and Merge).
   - Phase 3: Verification (Netlify check).
4. **Troubleshooting**: Handle "Build Failed" error.

### Output Constraints:
- Precise and Instructional tone.
- Professional SOP header.
```

## 2. Refunding a Customer (eCommerce)

```text
Act as a Process Quality Specialist. Generate an SOP for "Processing a Product Return Refund."

### Source Material:
Check the packing slip. If the item isn't damaged, go to Shopify. Find the order. Click Refund. Select the items. Click Refund $ total. Email the customer.

### SOP Requirements:
- Objective: Ensure accurate financial processing of returns.
- Prerequisites: Shopify Admin access.
- Troubleshooting: Customer paid with a gift card (special case).
- Definition of Success: Shopify marked as "Refunded."

### Output Constraints:
- Nested bullet points for sub-steps.
```
