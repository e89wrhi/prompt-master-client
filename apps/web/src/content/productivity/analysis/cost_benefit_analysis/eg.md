# Examples: Cost-Benefit Analysis

## 1. Migrating to a Cloud-Based Project Management Tool

```text
Act as a Senior Financial Analyst and Strategic Consultant. Conduct a rigorous Cost-Benefit Analysis for the following PROPOSAL: Migrating from localized Excel tracking to a enterprise-tier Jira/Asana license for the Engineering division.

### Financial Data & Context:
- Team size: 50 developers.
- Current cost: $0 (Excel).
- Current pain points: 15% of developer time lost to status updates and manual reporting.
- Proposed tool cost: $25/user/month.
- Implementation time: 2 weeks (requires 10% of senior dev time).

### Analysis Requirements:
1. **Hard Costs (Direct)**: List all immediate financial outlays including SaaS Licensing, Migration support, and Admin training.
2. **Soft Costs (Indirect)**: Identify hidden costs such as initial productivity dip during the learning curve and opportunity cost of senior devs during setup.
3. **Hard Benefits**: Quantify direct gains like a 10% reduction in status-meeting hours over a 12-month timeframe.
4. **Soft Benefits**: Evaluate intangible advantages like real-time visibility for stakeholders and reduced developer burnout.
5. **ROI Calculation**: Provide a projected ROI percentage and a "Break-Even Point" analysis.

### Output Constraints:
- Present the final comparison in a clearly formatted table.
- Conclude with a "Go/No-Go" recommendation based on the data.
- Use a Professional and Objective tone.
```

## 2. In-House Server vs. Managed Hosting

```text
Act as a Senior Financial Analyst and Strategic Consultant. Conduct a rigorous Cost-Benefit Analysis for the following PROPOSAL: Moving our data processing pipeline from an on-premise server rack to AWS Managed Services.

### Financial Data & Context:
- Current on-prem maintenance: $5k/month (Electricity, cooling, repairs).
- Server hardware refresh due next year: $60k estimated.
- AWS projected cost: $4k/month usage + $10k one-time migration fee.
- Uptime requirement: 99.99% (Current: 98.5%).

### Analysis Requirements:
1. **Hard Costs (Direct)**: List all immediate financial outlays including AWS monthly fees, migration consultants, and hardware decommissioning.
2. **Soft Costs (Indirect)**: Identify hidden costs such as staff retraining for AWS-specific architecture.
3. **Hard Benefits**: Quantify direct gains like the $60k capital expenditure avoidance over a 3-year timeframe.
4. **Soft Benefits**: Evaluate intangible advantages like improved disaster recovery and scalability for peak loads.
5. **ROI Calculation**: Provide a projected ROI percentage and a "Break-Even Point" analysis.

### Output Constraints:
- Present the final comparison in a table.
- Use a Professional tone.
```
