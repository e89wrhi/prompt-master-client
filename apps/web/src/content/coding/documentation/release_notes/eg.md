# Examples: Professional Release Notes

## 1. Major Release (v2.0.0): "The Performance Update"

```text
Act as a Product Manager. Generate the release notes for v2.0.0 of "DashboardPro."

### Context:
- **Focus**: Building for scale.
- **Highlights**: Rewrote the chart engine for 50FPS, reduced bundle size, introduced Auth0.
- **Breaking**: The old `/legacy/login` endpoint is removed.

### Requirements:
1. Provide a "From v1 to v2" migration snippet for the login logic.
2. Group the highlights into "Performance" and "Security."
3. Highlight the user-facing benefit: "Dashboard now loads in under 1 second."

### Output Constraints:
- Professional and Exciting tone.
```

## 2. Beta Patch (v1.2.0-beta): "Dark Mode Polish"

```text
Act as a DevRel Specialist. Generate notes for a beta update.

### Requirement:
- Focus on the community-requested "True Black" AMOLED theme.
- List 3 major bug fixes in the UI components.

### Output Constraints:
- Transparent and Welcoming tone.
```
